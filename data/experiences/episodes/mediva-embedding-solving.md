# 한국어 임베딩 모델 기반 벡터 검색을 통해 비정형 데이터 인용 정확도 개선

## Situation
의료기기 임상 프로토콜은 전문 용어와 긴 문맥이 섞인 비정형 데이터.
단순 키워드 매칭 사용 시 "심장 초음파 프로토콜" 같은 구체적 쿼리에 대해 일반적인 "심혈관 질환" 정보만 반환하는 검색 품질 저하 문제 존재.

## Action
한국어 의료기기 규제 도메인에 특화된 환경 구축을 통한 검색 정확도 향상 필요.
한국어 문장 유사도에 최적화된 ko-sroberta-multitask 모델을 임베딩 모델로 채택.
RecursiveCharacterTextSplitter로 Chunk Size=900, Overlap=100 설정하여 문맥 유지와 텍스트 분할 균형 확보.
FAISS 벡터 데이터베이스 구축 후, 청크 선택의 다양성을 고려하는 MMR 방식 대신 유사도 순으로 채택하는 Similarity Search 방식 적용, 가장 연관성 높은 상위 청크만 LLM이 참고하도록 구성.

## Result
추상적 질문에도 '교차설계', '단일기관', '심장판막역류증' 등 구체적 키워드를 벡터 데이터베이스에서 추출하여 인용하는 것을 확인.

## 이 에피소드의 핵심 메시지
"단순 키워드 검색의 한계를 극복하기 위해 한국어 특화 임베딩 모델과 최적화된 청킹 전략을 도입하여, 비정형 의료 데이터의 문맥을 유지하고 검색 정확도를 올린 RAG 고도화 경험."
