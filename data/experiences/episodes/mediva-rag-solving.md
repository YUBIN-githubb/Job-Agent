# 의료기기 법규 기반으로 답변하는 RAG 아키텍처 LLM을 이용해  할루네이션 해결 및 신뢰성 확보

## Situation
의료기기 임상시험 계획서는 법적 규제 근거 기반 작성이 필수.
의료기기 임상시험 관련 질문 시 기존 OpenAI LLM(GPT-3.5/4)에서 존재하지 않는 조항을 생성하거나 일반론적 답변만 제공하는 할루시네이션 발생.

## Action
한정된 도메인 지식 내에서만 답변을 생성하도록 제약하여 법적 근거가 확실한 팩트 기반 응답 시스템 구축 필요.
이를 위해 RAG Architecture 구현.
LangChain의 ConversationalRetrievalChain을 활용한 외부 지식 주입 파이프라인 구축.
추가적인 지침 제공과 페르소나 부여를 위해 Prompt Engineering 활용.
System Prompt에 Context 기반 답변 지시, 직접 인용 지시, Temperature 설정 명시.
chain_type으로 "stuff" 방식 채택, 검색된 모든 청크를 프롬프트에 주입하여 문맥 손실 최소화.

## Result
ChatGPT가 보인 할루시네이션 현상을 RAG 구조로 해결.
- ChatGPT는 "의료기기법 제17조"를 인용하며 임상시험 실시 기준을 명시한 법률이라고 답변. 그러나 실제 제17조는 "판매업 등의 신고"에 관한 내용으로 임상시험과 무관한 할루시네이션 답변.
- 반면 MEDIVA는 대상자 동의서 서명 절차(설명서 내용 제공 → 피험자 이해 → 자유의사 확인 → 서명 및 기록)를 단계별로 논리적으로 제시하여 임상시험 절차를 명확하게 안내. 법적 근거와 참고 문서를 명시하여 신뢰성 높은 정보 제공
- ChatGPT는 연장 신청 방법, 구비 서류, 수수료 여부 등 핵심 내용이 "기관마다 다를 수 있다"는 구체성이 부족한 답변으로, 임상시험 서류 관련 질문에 실질적 도움 불가.
- 반면 MEDIVA는 "사용기한 종료 전 신청, 종료 후 10일 이내 종료보고" 등 구체적 규정을 포함. 발급 기관을 "관할 지방청"으로 명시하고 승인 절차 및 결과 확인 방식까지 구체적으로 안내.

## 이 에피소드의 핵심 메시지
"RAG 아키텍처 도입과 프롬프트 엔지니어링을 통해 범용 LLM의 법률 할루시네이션 현상을 해결하고, 규제 준수 기반의 의료기기 특화 AI 챗봇을 구축한 경험."
