# MD5 해시 알고리즘을 활용한 벡터 데이터베이스 업데이트 시스템 최적화

## Situation
기존 시스템은 문서 하나 추가 시에도 전체 벡터 DB를 재생성해야 하는 구조. 데이터 양 증가에 따른 시스템 부하 가중 및 초기 구동 시간 증가 원인.

## Action
파일 고유의 MD5 Hash 값을 활용한 변경 감지 알고리즘 도입.
기존 해시값과 현재 파일 상태 값을 대조하여 변경 사항이 있는 파일만 선별적으로 재임베딩하는 로직 구현.

## Result
해시 기반 캐싱 전략으로 불필요한 중복 연산 제거.
전체 재임베딩 대비 업데이트 및 초기 로딩 속도 약 2배 단축, 사용자 경험 개선.

## 이 에피소드의 핵심 메시지
"MD5 해시 기반의 데이터 변경 감지 및 캐싱 전략을 도입하여, 벡터 DB의 불필요한 중복 임베딩 연산을 방지하고 업데이트 속도를 2배 단축한 경험."
