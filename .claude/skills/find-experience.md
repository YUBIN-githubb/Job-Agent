# experience-db: 경험 에피소드 검색 & 매칭

## 역할

자소서 문항이 요구하는 역량을 바탕으로 `data/experiences/episodes/` 아래
저장된 에피소드 중 가장 적합한 Top 3을 골라 반환한다.

**이 스킬은 글을 쓰지 않는다.** 초안 작성은 essay-draft가 담당한다.
이 스킬의 결과물은 반드시 "선정된 에피소드 ID + 선정 사유"의 형태다.

## 데이터 경로

- 에피소드 파일: `data/experiences/episodes/*.md`
- 프로젝트 배경 파일: `data/experiences/projects/*.md`
- 에피소드는 YAML 프런트매터 + STAR 본문 구조를 따른다
- 각 에피소드의 `source` 필드가 프로젝트 파일과 연결된다

## 에피소드 파일 스키마

```yaml
---
id: <고유 식별자, 파일명과 동일>
source: <프로젝트 파일 id, 없으면 생략>
primary_competency: <핵심 역량 1개>
tags: [역량태그, 맥락태그, 기술태그]
strength: <1-5, 본인이 느끼는 설득력>
used_count: <지금까지 자소서에 사용된 횟수>
last_used: <마지막 사용 회사/시기>
---
```

## 매칭 알고리즘

### 1단계: 입력 해석
요청에서 다음을 추출한다:
- 타겟 역량 키워드 (예: "리더십", "문제해결", "협업")
- 회사/직무 맥락 (있다면)
- 배제 조건 (예: "이번엔 캡스톤 말고")

### 2단계: 1차 필터링 (프런트매터 기반)
Glob으로 `data/experiences/episodes/*.md` 전체 파일 목록을 가져온 뒤,
각 파일의 YAML 프런트매터만 빠르게 읽어 다음 조건으로 필터링한다:

- `primary_competency`가 타겟 역량과 일치 → 가중치 3점
- `tags`에 타겟 역량 키워드 포함 → 가중치 2점
- `tags`에 관련 기술 스택 포함 → 가중치 1점
- `strength` ≥ 4 → 가중치 1점 추가
- `used_count` ≥ 3 → 가중치 1점 감점 (남용 방지)

### 3단계: 2차 판단 (본문 기반)
1차 필터에서 상위 5개 에피소드의 본문을 Read로 읽어들여,
문항의 구체적 의도와 에피소드의 STAR 서술 간 적합도를 판단한다.

판단 기준:
- 문항의 동사(이끌다/해결하다/설득하다)와 에피소드 Action의 일치도
- 결과(Result)의 구체성과 문항이 기대하는 "성과의 규모" 매칭
- 이미 다른 문항에 사용 예정인 에피소드는 회피 (중복 방지)

### 4단계: Top 3 반환

다음 형식으로 반환한다:

```markdown
## 매칭 결과 (타겟: [역량명])

### 1순위: [episode-id] (적합도: 높음)
- 선정 사유: [한 문장]
- 핵심 메시지: [에피소드의 주장]
- 주의점: [있다면, 예: "회사에 따라 조정 필요"]

### 2순위: [episode-id] (적합도: 중간)
...

### 3순위: [episode-id] (적합도: 보조)
...

## 다음 단계
essay-draft 스킬에 [1순위 episode-id]와 문항을 전달해 초안 작성.
```

## 특수 상황 처리

### 매칭 후보가 3개 미만일 때
- 2개: 2개만 반환하고 "세 번째 후보를 찾기 어렵다"고 명시
- 1개: 1개 반환하고 "에피소드 다양성이 부족하다. 새 에피소드 발굴 권장" 경고
- 0개: 검색 실패를 보고하고, `tags` 확장이나 에피소드 추가를 제안

### 중복 사용 경고
동일 에피소드가 같은 회사 자소서 내 다른 문항에 이미 배정된 경우,
자동으로 제외하고 그 사실을 사유에 명시한다.

### 프로젝트 배경이 필요할 때
선정된 에피소드의 `source` 필드가 있으면,
해당 프로젝트 파일의 경로도 반환해 essay-draft가 참조할 수 있게 한다.

## 사용 후 후처리

매칭이 확정되어 essay-draft로 넘어간 뒤에는,
해당 에피소드의 `used_count`를 1 증가시키고 `last_used`를 갱신한다.
(이 업데이트는 사용자 확인 후에만 수행한다.)

## 예시 호출

입력: "삼성전자 하반기 공채 '리더십 경험' 문항에 쓸 에피소드 찾아줘"

동작:
1. 타겟 역량 = "리더십" 추출
2. Glob: `data/experiences/episodes/*.md`
3. 프런트매터 기반 1차 필터링 (primary_competency=리더십 우선)
4. 상위 후보 본문 읽기 & STAR 적합도 판단
5. Top 3 반환
