# web-research 스킬

## 역할

회사명과 직군을 입력받아 웹에서 해당 기업의 최신 정보를 조사하고,
구조화된 원시 조사 결과를 반환한다.

**입력**: 회사명, 직군(또는 공고 정보), 선택적 추가 키워드
**출력**: 카테고리별로 분류된 원시 사실 목록 + 출처 URL

**이 스킬은 해석하지 않는다.** "이 회사는 혁신적이다" 같은 평가는
company-report의 영역이다. 여기서는 **확인된 사실과 1차 소스**만 모은다.

## 조사 대상 카테고리

자소서 재료로 유용한 정보를 다음 7개 카테고리로 나눠 조사한다.
모든 카테고리를 채울 필요는 없다. 발견되는 만큼만 수집한다.

### 1. 회사 개요 (Company Overview)
- 정식 회사명, 설립 연도, 본사 위치
- 주요 사업 영역 (무엇을 만들고 파는 회사인가)
- 규모 (직원 수, 매출, 상장 여부)
- 대표이사/창업자 (최근 변경 여부 확인)

### 2. 최근 전략 방향 (Strategic Direction)
가장 중요한 카테고리. 최근 1년 이내 한정:
- 신사업/신제품 발표
- 기존 사업 확장/축소
- 조직 개편이나 리더십 변경
- CEO 신년사 / IR 보도자료 / 공식 블로그의 핵심 메시지
- "미래 먹거리"로 언급되는 영역

### 3. 최근 뉴스 & 이슈 (Recent News)
최근 3~6개월 내:
- 주요 수주/수상/인수합병
- 기술 발표나 제품 출시
- 업계 내 이슈 (규제, 경쟁사 대비 위치)
- 긍정 이슈 우선, 리스크 이슈는 "파악 필요" 플래그로 표시

### 4. 직군/팀 관련 정보 (Role Context)
공고의 직군(job_group)에 특화:
- 해당 부서의 최근 활동
- 팀 블로그, 기술 블로그 게시물
- 채용 담당자나 팀장의 공개 글/인터뷰
- 해당 직군에서 이 회사가 쓰는 주요 기술/방법론

### 5. 제품/서비스 (Products)
- 대표 제품/서비스명과 간단한 설명
- 최근 업데이트나 출시된 신규 제품
- 사용자 규모나 시장 점유율 (공개된 경우)

### 6. 인재상 & 문화 (Culture & Values)
- 공식 인재상 / 핵심 가치 (채용 페이지 기준)
- 복지/근무 환경 특이점
- 현직자 리뷰 요약 (잡플래닛, 블라인드 등 - 출처 주의)
- CEO 인터뷰에서 강조되는 자질

### 7. 업계 맥락 (Industry Context)
- 주요 경쟁사 3~5개
- 이 회사의 차별화 포인트 (자체 주장 + 업계 평가)
- 해당 업계의 현재 트렌드

## 검색 쿼리 설계 원칙

### 일반 원칙
- 영어 + 한국어 혼합 쿼리 (한국 기업도 영문 자료가 더 풍부한 경우 있음)
- "2026", "최근", "latest" 같은 시간 한정어 활용
- 너무 짧은 쿼리 피하기 (노이즈 많음)

### 카테고리별 쿼리 예시
회사명이 "삼성전자"일 때:

**최근 전략 방향**:
- `"삼성전자" 신년사 2026`
- `"삼성전자" 2026 전략 보도자료`
- `Samsung Electronics 2026 strategy announcement`
- `"삼성전자" CEO 메시지`

**최근 뉴스**:
- `"삼성전자" 최근 뉴스 -사건 -사고` (부정 이슈 제외 옵션)
- `Samsung Electronics announcement 2026`

**직군 관련** (반도체 엔지니어라면):
- `"삼성전자" 반도체 기술 블로그`
- `"삼성전자" 파운드리 2026`
- `Samsung foundry 3nm 2nm 로드맵`

**인재상**:
- `"삼성전자" 채용 인재상`
- `"삼성전자" 핵심 가치`
- `"삼성전자" 신입 공채 자소서 후기` (실제 지원자 관점)

### 쿼리 수 제한
카테고리당 2~3개 쿼리로 제한한다. 너무 많이 검색하면:
- 시간 낭비
- 중복 결과가 많아짐
- 오히려 노이즈 증가

총 검색 횟수: **15~20회 이내**로 제한.

## 출처 품질 기준

발견된 정보의 출처를 다음 4단계로 분류하고, 우선순위 높은 것만 사용한다.

### Tier 1 (최우선 사용)
- 회사 공식 웹사이트, 공식 블로그
- 공식 보도자료
- 회사 공식 SNS 계정 (검증 가능한 경우)
- SEC/금감원 공시자료

### Tier 2 (신뢰도 높음)
- 주요 언론사 기사 (조선일보, 중앙일보, 한겨레, 매일경제, 한국경제 등)
- 업계 전문 매체 (전자신문, 디지털데일리, TechCrunch, The Verge)
- 애널리스트 리포트 (공개된 경우)

### Tier 3 (보조 참고)
- 중소 언론 기사
- 기술 블로그 (Medium, velog, GitHub)
- Wikipedia (검증용, 단독 소스로는 불가)

### Tier 4 (주의하여 사용)
- 잡플래닛, 블라인드 등 커뮤니티 리뷰
- 개인 블로그
- Reddit, 나무위키

Tier 4 정보는 반드시 "현직자 의견" 같은 맥락과 함께 표시하고,
팩트가 아닌 견해임을 명시한다.

## 특수 상황 처리

### 회사 정보가 거의 없는 경우
(소규모 스타트업, 외국계 소규모 지사)

- 확장 검색: CEO 이름으로 검색 (CEO 인터뷰, LinkedIn 등)
- 투자 정보 검색: `"{회사명}" 투자 유치`, `"{회사명}" 시리즈`
- 제품 역검색: 회사의 대표 제품으로 역으로 회사 정보 확보
- 그래도 부족하면: 출력에 "정보 제한적" 플래그 + 사용자에게 내부 정보 보완 요청

### 동명 회사 구분
"삼성" 같은 경우 계열사가 많아 혼동 가능:
- 공고에 명시된 정식 명칭 기준으로만 조사
- 검색 시 정확한 명칭을 따옴표로 묶어 사용 (`"삼성전자"`)
- 결과가 다른 계열사 정보인지 의심되면 제외

### 외국 회사
- 회사명의 영문 표기를 정확히 확인
- 한국 지사가 있는 경우 본사와 한국법인을 분리해서 조사
- 최근 뉴스는 현지 언론 + 한국 언론 모두 확인

### 공공기관
- 공식 웹사이트, 국회 국정감사 자료, 감사원 보고서
- 담당 부처 장관의 최근 지시사항/정책 방향
- NCS 직무기술서 (www.ncs.go.kr)

### 검색 결과가 오래된 경우
최신순으로 정렬했는데도 1년 이상 된 결과만 나오면:
- 해당 주제에 대한 최근 활동이 없다는 의미
- 그 사실 자체를 리포트에 포함 ("최근 공개 활동 제한적")
- 다른 카테고리로 조사 방향 전환

## 작업 순서

1. 입력 검증 (회사명 정확성 확인)
2. 카테고리별 검색 쿼리 설계 (2~3개씩)
3. WebSearch 및 WebFetch 순차 실행
4. 각 결과의 출처 Tier 판정
5. 카테고리별로 정리 및 중복 제거
6. 출처 URL과 함께 원시 결과 구성
7. 조사 한계 섹션 작성
8. company-report 스킬에 전달할 수 있는 형식으로 반환

## 주의사항

- 이 스킬은 **조사만** 한다. "자소서에 이렇게 쓰세요" 같은 조언이나
  재구성은 절대 하지 않는다.
- 확인되지 않은 루머, 개인 추측, 출처 불분명한 정보는 포함하지 않는다.
- 검색 결과가 많이 나온다고 모두 포함하지 않는다. 자소서 작성에
  실제로 도움 되는 "구체적 사실" 위주로 선별한다.
- Tier 4 소스는 반드시 명시적으로 표시한다.
- 검색 총량은 15~20회로 제한 (시간/자원 효율).
