# web-scraping 스킬

## 역할

입력(URL 또는 텍스트)을 받아 채용공고의 "깨끗한 본문 텍스트"만 만든다.
회사명·직무·마감일 같은 필드 추출은 이 스킬의 일이 아니다.

**입력 형태**: URL 문자열 또는 공고 텍스트 (또는 둘 다)
**출력 형태**: 정제된 마크다운 본문 + 메타 정보

## 수집 전략 (우선순위 순)

### 1. 사용자 직접 입력 (최우선)
사용자가 공고 텍스트를 붙여넣은 경우:
- WebFetch를 시도하지 않는다
- 바로 정제 단계로 진행
- 가장 안정적이며 법적 리스크 없음

### 2. WebFetch (내장 도구)
URL만 주어진 경우, 다음 순서로 진행:

1. **robots.txt 확인**:
   - `{domain}/robots.txt`를 WebFetch로 가져온다
   - 채용 공고 경로가 `Disallow`에 포함되어 있으면 중단
   - 중단 시 사용자에게 수동 붙여넣기 요청

2. **본문 수집**:
   - WebFetch로 URL 가져오기
   - 응답 크기 1KB 미만 → 차단 페이지로 간주, 실패 처리
   - "JavaScript가 필요합니다" 포함 → JS 렌더링 필요, 실패 처리
   - 로그인 페이지로 리다이렉트 감지 → 실패 처리

3. **실패 시**:
   - User-Agent 위장 등 우회 트릭은 절대 사용하지 않는다
   - 사용자에게 "자동 수집이 차단되었습니다. 브라우저에서 공고 본문을
     복사해 붙여넣어 주세요"라고 안내

### 3. Playwright MCP (선택적)
프로젝트에 Playwright MCP가 설치되어 있고 사용자가 명시적으로 요청한
경우에만 사용. 기본 활성화하지 않는다.

## 본문 정제 규칙

원시 HTML/텍스트를 받으면 다음 순서로 정제한다:

### 제거 대상
- 상단 네비게이션, 메뉴바
- 하단 푸터, 저작권 표시
- 쿠키 알림 배너
- "이 공고를 본 사람들이 본 다른 공고" 같은 추천 섹션
- 소셜 공유 버튼, 스크롤 유도 문구
- 연속된 빈 줄 (최대 1줄로 압축)

### 보존 대상 (중요)
한국 채용공고의 핵심 섹션은 반드시 그대로 유지한다:

- 회사 소개 / 기업 정보
- 직무 개요 / 포지션
- 담당 업무 / 주요 업무
- 자격 요건 / 필수 자격 (정규직 필수 요건)
- 우대 사항
- 근무 조건 / 근무지 / 급여
- 전형 절차 / 채용 과정
- 접수 기간 / 마감일
- **자기소개서 문항** (가장 중요 - 다음 파이프라인의 핵심 입력)

섹션 제목(## 또는 **볼드**)은 원본 형식을 유지한다.

## 출력 형식

정제 완료 후 다음 형식으로 반환한다:

```markdown
---
collection_method: direct_input | webfetch | playwright
source_url: https://... (있다면)
collected_at: 2026-04-16T10:30:00+09:00
confidence: high | medium | low
text_size: 12340
warnings:
  - "자소서 문항이 '더 보기' 버튼 뒤에 있어 누락 가능성"
---

# 원시 공고 본문

[정제된 텍스트 - 원본 섹션 구조 유지]
```

## 신뢰도 판정

- **high**: 사용자 직접 입력 / WebFetch 성공 + 본문 2KB 이상 + 주요 섹션 검출됨
- **medium**: 본문은 있으나 주요 섹션 중 일부 누락 / Playwright MCP 사용
- **low**: 본문이 지나치게 짧음 / 섹션 구조가 거의 없음

신뢰도가 `low`면 data-parser로 넘기기 전에 사용자에게 수동 보완을 먼저
요청해야 한다.

## 특수 상황 처리

### 봇 차단 감지
응답에 다음 중 하나가 포함되면 차단으로 판정하고 중단한다:
- Cloudflare 차단 페이지
- reCAPTCHA 스크립트
- "Access Denied", "봇으로 의심됩니다" 같은 메시지
- HTTP 403, 429 상태

재시도하지 않고 사용자에게 수동 입력을 요청한다.

### 로그인 필수 사이트
링크드인, 일부 대기업 공채 사이트 등 로그인이 필요한 경우:
- 사용자에게 "로그인 후 공고 본문을 복사해 붙여넣어 주세요" 안내
- 절대 사용자 자격증명을 요구하지 않는다

### URL이 이미지/PDF인 경우
확장자가 `.pdf`, `.png` 등이면:
- WebFetch로 내용을 직접 추출하려 하지 않는다
- 사용자에게 "이미지/PDF 공고는 직접 텍스트를 추출해 붙여넣어 주세요"
  라고 안내

### robots.txt 자체를 못 가져오는 경우
robots.txt 요청이 실패하면 보수적으로 접근:
- 해당 도메인이 사람인/잡코리아/링크드인 등 알려진 대형 사이트면 차단으로 간주
- 기업 자체 채용 페이지면 허용 가정 (일반적으로 공개 공고)

## 작업 순서

1. 입력 타입 판별 (URL / 텍스트 / 둘 다)
2. URL이면 robots.txt 체크
3. 본문 수집 (수동 입력 > WebFetch)
4. 정제 (제거/보존 규칙 적용)
5. 신뢰도 판정
6. 출력 형식으로 포장
7. data-parser에 전달 가능한 상태로 반환

## 주의사항

- 이 스킬의 출력은 **원시 텍스트** 수준이다. 회사명, 마감일 같은
  "해석된 정보"를 여기서 뽑지 않는다.
- 자소서 문항 섹션을 잃으면 전체 파이프라인이 무너지므로,
  "자기소개서", "자소서", "항목" 키워드가 본문에 있는지 최종 확인한 뒤
  없으면 `warnings`에 명시한다.
- 사용자의 로그인 정보나 개인 인증 토큰을 절대 요구하지 않는다.
