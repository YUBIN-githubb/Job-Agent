# competency-map 스킬

## 역할

question-decoder가 분석한 "평가 의도"와 "평가 축 키워드"를 받아,
경험 에피소드 DB(`data/experiences/episodes/*.md`)의 태그 체계와
같은 언어로 번역한다.

**입력**: question-decoder의 출력 (평가 축 키워드, 문항 유형)
      + company-report의 인재상 (있다면)
**출력**: 역량 태그 매핑 + find-experience 검색 명세

**이 스킬은 경험을 검색하지 않는다.** 검색 키워드만 만들고,
실제 매칭은 find-experience가 수행한다.

## 표준 역량 사전

에피소드 프런트매터의 `tags`, `primary_competency`와 일치해야 한다.
아래 7개 카테고리와 하위 태그가 표준이다.

### 1. 리더십 / 조직관리
`리더십`, `팀장경험`, `역할분배`, `동기부여`, `의사결정`, `위임`,
`목표설정`, `성과관리`

### 2. 협업 / 커뮤니케이션
`협업`, `팀워크`, `갈등관리`, `설득`, `경청`, `이해관계조율`,
`피드백수용`, `다양성존중`

### 3. 문제해결 / 분석
`문제해결`, `원인분석`, `데이터기반판단`, `가설검증`, `창의적접근`,
`트레이드오프`, `우선순위판단`, `논리적사고`

### 4. 도전 / 실패극복
`도전정신`, `실패경험`, `회복탄력성`, `한계돌파`, `처음시도`,
`불확실성대응`, `피벗`, `재도전`

### 5. 자기개발 / 학습
`자기주도학습`, `역량개발`, `자격증`, `독학`, `멘토링`,
`습관형성`, `꾸준함`, `성장마인드셋`

### 6. 실행력 / 주도성
`주도성`, `기획`, `실행력`, `프로세스개선`, `자동화`,
`일정관리`, `자원관리`, `결과도출`

### 7. 윤리 / 책임감
`책임감`, `윤리의식`, `정직`, `원칙준수`, `사회적책임`,
`고객지향`, `봉사`, `공정성`

### 맥락 태그 (역량이 아닌 배경 정보)
**경험 유형**: `인턴`, `대외활동`, `학교프로젝트`, `동아리`,
`아르바이트`, `공모전`, `연구`, `창업`, `봉사활동`, `해외경험`, `군경험`

**기술 스택**: `Python`, `Java`, `SQL`, `Excel`, `데이터분석`,
`머신러닝`, `회계`, `마케팅`, `기획`, `설계`, `CAD`, `회로설계`

## 매핑 규칙

### 문항 키워드 → 역량 태그 번역표

| question-decoder 평가 축 키워드 | primary 후보 | secondary 후보 |
|---|---|---|
| 역경 극복, 어려움, 실패 | `실패경험`, `회복탄력성` | `도전정신`, `문제해결` |
| 도전, 새로운 시도 | `도전정신`, `처음시도` | `주도성`, `불확실성대응` |
| 이끌다, 팀장, 조직 | `리더십`, `팀장경험` | `역할분배`, `동기부여` |
| 갈등, 의견 충돌, 설득 | `갈등관리`, `설득` | `협업`, `이해관계조율` |
| 해결, 분석, 원인 파악 | `문제해결`, `원인분석` | `논리적사고`, `데이터기반판단` |
| 협력, 함께, 팀 | `협업`, `팀워크` | `경청`, `피드백수용` |
| 주도, 먼저, 자발적 | `주도성`, `실행력` | `기획`, `결과도출` |
| 학습, 성장, 부족함 인지 | `자기주도학습`, `역량개발` | `성장마인드셋`, `꾸준함` |
| 윤리, 원칙, 정직 | `윤리의식`, `책임감` | `원칙준수`, `공정성` |
| 지원 동기, 관심, 열망 | (역량 아님 — 아래 "지원동기 특례" 참조) | |
| 강점, 잘하는 것 | (역량 아님 — 아래 "강점약점 특례" 참조) | |

### primary / secondary 판정

하나의 문항에서 최대 2개의 primary, 3개의 secondary를 추출한다.

**primary 기준**: 문항의 핵심 동사/주제어와 직접 연결되는 역량
**secondary 기준**: 좋은 답변을 위해 간접적으로 필요한 역량

예시:
- "팀을 이끌며 갈등을 해결한 경험"
  - primary: `리더십`, `갈등관리`
  - secondary: `설득`, `의사결정`, `팀워크`

- "가장 도전적이었던 경험과 배운 점"
  - primary: `도전정신`, `실패경험`
  - secondary: `회복탄력성`, `자기주도학습`, `성장마인드셋`

### 지원동기 문항 특례

지원동기 문항은 역량이 아니라 **정보**를 평가한다.
이 경우 역량 태그 대신 다음을 반환한다:

```yaml
type: 지원동기
primary_tags: []
secondary_tags: []
required_sources:
  - company_report (필수)
search_strategy: 역량 매칭 대신 company-report의 "자소서 활용 포인트" 섹션 참조
note: find-experience가 아닌 company-report 기반으로 작성해야 함
```

### 강점약점 문항 특례

강점약점 문항은 역량 자체가 아니라 **자기 인식**을 평가한다.
강점 부분은 직무 관련 역량 태그를, 약점 부분은 별도로 처리한다:

```yaml
type: 강점약점
strength_tags: [직무 연관 역량 2~3개]
weakness_strategy: "실제 약점 + 인지 + 구체적 대응 노력" 구조
note: 강점은 find-experience로 에피소드 검색, 약점은 별도 서술
```

## 회사 인재상 가중치

company-report의 "문화와 일하는 방식" 섹션에 인재상이 있으면,
매칭 결과에 가중치를 적용한다.

### 적용 방식

인재상 키워드를 표준 역량 사전에 매핑한 뒤,
해당 역량이 primary/secondary 후보에 있으면 우선순위를 올린다.

예:
- 인재상 "Co-prosperity" → `협업`, `팀워크`, `이해관계조율`
- 문항 분석에서 `협업`이 secondary였다면 → primary로 승격 고려

### 가중치 적용 원칙
- 인재상 매칭은 **보조 기준**이다. 문항의 직접 요구를 뒤집지 않는다.
- "리더십을 묻는 문항"에 인재상이 "협업"이라고 해서 `협업`을
  primary로 바꾸지 않는다. secondary에 추가하는 정도.
- 인재상이 너무 추상적이면 ("열정", "창의") 가중치 적용 불가로 판정.

## 출력 형식

각 문항에 대해 다음 형식으로 반환한다:

```markdown
## 문항 [N] 역량 매핑

### 문항 원문
> [원문 그대로]

### 역량 매핑 결과
- **primary**: [`태그1`, `태그2`]
- **secondary**: [`태그3`, `태그4`, `태그5`]
- 인재상 가중치 반영: [있다면 어떤 태그가 어떻게 조정되었는지]

### find-experience 검색 명세
```yaml
search_query:
  primary_competency: [태그1, 태그2]  # OR 조건
  tags_include: [태그3, 태그4]         # 보조 필터
  tags_exclude: []                     # 제외할 태그
  preferred_experience_type: [인턴, 학교프로젝트]  # 있다면
  preferred_tech_stack: []             # 직무 관련 기술 (있다면)
```

### 에피소드 선택 가이드
- 이상적 에피소드: [어떤 특성의 경험이 가장 잘 맞는지, 1~2문장]
- 피해야 할 에피소드: [이 문항에 안 맞는 경험 유형, 1문장]

### 다른 문항과의 관계
- [N번 문항]과 역량 겹침 여부: [있다면 경고 + 차별화 전략]
```

## 전체 분석 조립

모든 문항의 매핑이 끝나면, 전체를 하나로 조립한다:

```markdown
---
job_id: samsung-electronics-semi-designer-2604
company: 삼성전자
position: 반도체 설계 엔지니어
analyzed_at: 2026-04-16T11:00:00+09:00
question_count: 2
company_report: data/reports/samsung-electronics-2604.md
---

# 문항 분석 결과

[...]
### 역량 매핑 (competency-map)
[...]

## 문항 2 분석

```

## 에피소드 DB 커버리지 사전 체크

모든 문항의 역량 태그가 확정된 뒤, `data/experiences/episodes/*.md`를
Glob + 프런트매터 읽기로 빠르게 스캔해 다음을 확인한다:

- 각 primary 태그를 `primary_competency`로 가진 에피소드가 있는가
- 각 secondary 태그를 `tags`에 포함한 에피소드가 있는가
- 태그는 있지만 `strength`가 3 이하인 경우 "약한 에피소드" 경고

커버리지 결과를 "전체 요약" 섹션에 포함한다.
부족한 태그가 있으면 "새 에피소드 정리가 필요합니다" 안내.

## 특수 상황 처리

### decoder가 복합형으로 분류한 문항
하위 질문 각각에 대해 별도 매핑을 수행한 뒤, 통합:

```yaml
sub_questions:
  - sub: "경험을 서술하시오"
    primary: [도전정신]
    secondary: [실행력]
  - sub: "배운 점"
    primary: [자기주도학습]
    secondary: [성장마인드셋]
combined_primary: [도전정신, 자기주도학습]
combined_secondary: [실행력, 성장마인드셋]
```

### 역량 사전에 없는 키워드가 나온 경우
decoder가 표준 사전에 없는 평가 축을 제시하면:
- 가장 가까운 표준 태그로 매핑하고 사유를 명시
- 예: "글로벌 감각" → `해외경험` (맥락 태그) + `다양성존중` (역량 태그)

### 같은 역량이 3개 이상 문항에서 primary인 경우
essay-writer가 에피소드 중복 없이 쓰기 매우 어렵다는 신호:
- 2번째 문항부터는 같은 역량의 **다른 측면**을 secondary로 접근 권장
- 예: 1번 = 리더십(역할분배), 3번 = 리더십(동기부여) → 같은 역량이지만
  하위 태그로 차별화

## 작업 순서

1. question-decoder 출력의 평가 축 키워드 수령
2. 번역표 적용하여 primary/secondary 후보 도출
3. company-report 인재상 가중치 적용 (있다면)
4. primary/secondary 최종 확정
5. find-experience 검색 명세 생성
6. 에피소드 선택 가이드 작성
7. 전체 문항 분석 후 교차 검증
8. 에피소드 DB 커버리지 사전 체크
9. 전체 분석 문서 조립
10. question-analyst에 반환

## 주의사항

- 표준 역량 사전의 태그만 사용한다. 임의로 새 태그를 만들지 않는다.
  (에피소드 DB의 태그와 불일치하면 검색이 안 된다)
- 인재상 가중치가 문항의 직접 요구를 뒤집지 않도록 주의.
- 매핑 결과가 너무 많으면 (primary 3개 이상) 과도한 것이다. 좁혀라.
- 지원동기/강점약점 문항은 역량 매핑이 아닌 특례 처리를 따른다.
- 에피소드 DB 커버리지 체크는 **읽기 전용**이다. 에피소드를 수정하지 않는다.
